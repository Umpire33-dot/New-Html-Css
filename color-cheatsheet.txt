Color Cheatsheet — formats, usage, examples, and accessibility tips

1) Named color keywords
- Purpose: Simple, human-readable names (limited set).
- Examples: red, blue, lightblue, darkslategray, transparent
- Usage: color: blue; background-color: lightblue;

2) Hexadecimal (HEX)
- Purpose: Common compact color notation: #RRGGBB or shorthand #RGB
- Examples: #ff0000 (red), #00ff00 (lime), #03f (short for #0033ff)
- 8-digit hex with alpha: #RRGGBBAA (e.g., #00000080 for black at 50%)
- Usage: color: #333333;

3) RGB / RGBA
- Purpose: Red/Green/Blue channels; `rgba()` adds alpha for transparency.
- Examples: rgb(255,0,0) ; rgba(255,0,0,0.5) /* 50% transparent red */
- Values: 0–255 or 0%–100% for each channel; alpha 0–1.
- Usage: background-color: rgba(0,0,0,0.6);

4) HSL / HSLA
- Purpose: Hue-Saturation-Lightness (often easier to tweak than RGB).
- Examples: hsl(200, 80%, 50%); hsla(200, 80%, 50%, 0.6)
- Hue: 0–360 (degrees), Saturation/Lightness: percentages.
- Usage: color: hsl(210, 15%, 20%);

5) CSS Color Module 4 features
- `color()` and `lab()`/`lch()` for wide-gamut colors (advanced).
- 8-digit hex and `rgb()` with space-separated syntax: rgb(255 0 0 / 50%)
- Example: color: rgb(255 0 0 / 0.5); /* red at 50% alpha */

6) System / special keywords
- `currentColor`: uses the element's `color` value (useful for borders/icons).
- `transparent`: fully transparent color.
- Example: border: 2px solid currentColor;

7) CSS Variables for colors (recommended)
- Purpose: Centralize palette, easy theming.
- Example:
  :root {
    --bg: #f8f9fa;
    --text: #222222;
    --primary: #0077cc;
  }
  body { background-color: var(--bg); color: var(--text); }

8) Gradients
- Purpose: Smooth transitions between colors; no images needed.
- Examples:
  background: linear-gradient(90deg, #ff7a18, #af002d 70%);
  background: radial-gradient(circle at 20% 20%, #fff, #ddd);

9) Opacity vs RGBA/HSLA
- `opacity` affects entire element (including children).
- `rgba()`/`hsla()` affects only the color; preferred for translucent backgrounds.
- Example: use background-color: rgba(0,0,0,0.4) instead of opacity: 0.4 when you don't want children faded.

10) Contrast & Accessibility (WCAG)
- Aim for contrast ratios:
  - 4.5:1 for normal text (WCAG AA)
  - 3:1 for large text (WCAG AA)
  - 7:1 for stricter (WCAG AAA)
- Tools: WebAIM Contrast Checker, browser devtools color picker with contrast.
- Tip: test combinations of text color vs background (including translucent overlays).

11) Choosing palettes & best practices
- Start with a neutral base (grays) + primary + accent + success/warning/danger colors.
- Use `--` variables for semantic names: --color-bg, --color-text, --color-primary.
- Limit palettes to 4–6 core colors for clarity.

12) Responsive / theme switching (example)
- Light/dark mode using variables:
  :root { --bg: #fff; --text: #111; }
  @media (prefers-color-scheme: dark) {
    :root { --bg: #111; --text: #eee; }
  }

13) Quick conversions & tips
- HEX → RGB: split bytes: #RRGGBB → R,G,B decimal.
- Prefer `rem`/`em` for scalable UI, but colors are unitless.
- Use alpha with `rgba()`/`hsla()` or the slash syntax: rgb(0 0 0 / 0.5).

14) Common mistakes to avoid
- Using named colors for precise branding (use hex/rgba for consistency).
- Using `opacity` when you mean only background translucency.
- Poor contrast (low readability).

15) Useful links & tools
- WebAIM Contrast Checker: https://webaim.org/resources/contrastchecker/
- MDN Color documentation: https://developer.mozilla.org/en-US/docs/Web/CSS/color
- Color palette generators: https://coolors.co/, https://color.adobe.com/
